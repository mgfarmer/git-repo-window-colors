‚≠ê Starred Keys Guide

The Starred tab provides a convenient way to collect and manage your most frequently used VS Code theme keys in one place. Instead of searching through multiple tabs (Titlebar, Activitybar, Statusbar, etc.) to find the elements you customize most often, you can star them and access them all in a single dedicated tab.

What is the Starred Tab?

When working with profiles, VS Code provides hundreds of theme keys that control different parts of the interface. The Starred tab is your personal collection of keys you use most frequently. It acts as a quick-access panel for customizing the elements you care about most.

üí° Tip: The Starred tab is especially useful when you have a set of keys you modify across multiple profiles. Instead of navigating through multiple tabs each time, you can access all your favorite keys in one place.

How to Star Keys

Starring a key is simple and can be done from any profile mapping tab:

  1. Open a profile in the Profiles tab
  2. Navigate to any mapping tab (Titlebar, Activitybar, Statusbar, etc.)
  3. Find the theme key you want to star
  4. Click the ‚òÜ (empty star) icon next to the key name
  5. The star will become filled ‚òÖ, indicating it's starred

The key will now appear in the Starred tab for easy access.

Managing Starred Keys

Viewing Starred Keys

Click on the ‚òÖ Starred tab in the profile mapping section to see all your starred keys in one place. The tab badge shows how many keys you've starred.

Unstarring Keys

To remove a key from your starred collection:

  1. Go to the Starred tab (or find the key in any other tab)
  2. Click the ‚òÖ (filled star) icon next to the key
  3. The star will become empty ‚òÜ, and the key will be removed from the Starred tab
Note: Unstarring a key does NOT remove its color mapping. It only removes the key from the Starred tab. The color mapping remains active in the profile.

Assigning Colors to Starred Keys

Once you have keys starred, you can assign colors to them just like in any other tab:

  1. Select a Palette Slot from the dropdown (Primary, Secondary, Accent1-3, etc.)
  2. Optionally adjust the Opacity slider to make the color more or less transparent
  3. Or use Fixed Color to assign a specific color independent of the palette

Understanding the Starred Tab Badge

The badge on the Starred tab provides quick insights into your color mappings:

  • Normal Badge (Number): Shows the total number of starred keys that have colors assigned
  • ‚ö†Ô∏è Warning Badge (X/Y): Appears when you have uncolored starred keys
    • X = Number of starred keys with colors
    • Y = Number of starred keys without colors
    • The badge turns yellow to draw attention
Example:
A badge showing ‚ö†Ô∏è 5/3 means you have 5 starred keys with colors assigned and 3 starred keys that don't have colors yet.

Uncolored Key Warnings

When viewing the Starred tab, any keys that don't have colors assigned will display a warning indicator:

  • A ‚ö†Ô∏è warning icon appears before the dropdown
  • This visual reminder helps you identify which starred keys still need colors
  • The warning disappears once you assign a color to the key
üí° Pro Tip: Use the warning indicators to quickly identify incomplete mappings in your starred collection. This is especially helpful when setting up a new profile or ensuring all your important elements are colored.

Starred Keys Across Profiles

Your starred keys collection is global and persists across all profiles. This means:

  • Keys starred in one profile will appear in the Starred tab of all profiles
  • You only need to star a key once, and it's available everywhere
  • Different profiles can have different colors for the same starred keys
Example Workflow:
You might star titleBar.activeForeground, activityBar.background, and statusBar.background as your core elements. These will appear in the Starred tab for all your profiles, but you can assign different colors in your "Dark Theme" profile versus your "Light Theme" profile.

Empty Starred Tab

If you haven't starred any keys yet, the Starred tab will display a helpful message:

"No starred keys yet. Click the star icon next to any mapping key to add it here."

Simply navigate to any other tab and start starring the keys you use most frequently!

Best Practices

  • Star Your Essential Keys: Focus on the 10-20 keys you modify most often across different profiles
  • Keep It Organized: The Starred tab shows keys in the same order they appear in the regular tabs, grouped by section
  • Use for Quick Edits: When fine-tuning colors, the Starred tab lets you adjust your core elements without navigating between multiple tabs
  • Monitor Warnings: Check the badge warning to ensure all your important keys are colored

Common Use Cases

Creating a Consistent Look

Star the primary keys you want to customize (titlebar, activitybar, statusbar backgrounds) and quickly apply your palette across all of them from the Starred tab.

Experimenting with Colors

Star a set of related keys (like all editor group borders) and experiment with different slot assignments and opacities in one view.

Maintaining Multiple Profiles

When you have several profiles (work, personal, presentation), star your core keys and use the Starred tab to quickly configure each profile's version of those elements.
