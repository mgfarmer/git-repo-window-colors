Getting Started with Git Repo Window Colors

Git Repo Window Colors helps you visually distinguish different Git repositories and branches by applying custom colors to your VS Code window. This is especially useful when working with multiple projects or switching between different branches frequently.

Quick Start: Simple Mode

We recommend starting with Simple Mode to get familiar with the extension's core features. Simple mode lets you define color rules for repositories and branches using straightforward color specifications.

Creating Your First Rules

  1. Open the Rules tab in the configuration panel
  2. Click "+ Add Repo Rule" to create a repository-based rule
  3. Enter a repository identifier (name, URL pattern, or path fragment)
  4. Choose colors for your window elements (titlebar, activitybar, statusbar)
  5. Click "+ Add Branch Rule" to add branch-specific coloring
Example Repository Rule:
my-project : #2E7D32
This applies green tones when the repository name contains "my-project"
Example Branch Rule:
bugfix : blue
This applies blue tones when on the bugfix branch. All other branches remain unchanged. This rule accepts regular expressions so you can match multiple branch names if needed. You can also use multiple branch rules to match different branch naming conventions.
Pro Tip: Configure your colors with the target repository open in your workspace! This allows you to see real-time changes to the full VS Code UI as you adjust colors. You'll immediately see how titlebar, activitybar, statusbar, and editor tabs look with your chosen colors, making it much easier to fine-tune your color scheme. You can also turn on "Preview Selected Rules" in the Rules tab to see changes without needing to switch repositories or branches.

Understanding the Configuration Panel

The configuration panel has these tabs:

  • Rules: Create and manage repository and branch color rules
  • Branch Tables: Manage shared branch tables
  • Profiles: Define reusable color palettes with advanced mapping options
  • Color Report: View exactly which colors are currently applied and why
  • Starred: View all your starred elements in one tab for easy application of colors

Key Settings

Under "Other Settings" you'll find options to control which UI elements get colored:

Note: These settings only apply when using simple colors. When using Profiles, these color-related settings are controlled by the profile configuration. The currently selected rule is using a profile.
  • Color Status Bar: Apply colors to the bottom status bar
  • Color Editor Tabs: Colorize the background of editor tabs
  • Color Inactive Titlebar: Color the titlebar even when VS Code is not focused
  • Color Knob: Choose intensity of primary color applied to the non-TitleBar elements (tabs and activity bar)

Testing Your Configuration

  1. Create a rule matching your current repository or branch
  2. Observe the color changes in real-time as you edit
  3. Use the Color Report tab to see which theme keys are being colored
  4. Adjust colors until you're satisfied with the result
Helpful Debugging: If colors aren't appearing as expected, check the Color Report to see which rules are matching and what colors are being applied. This is invaluable for troubleshooting rule patterns.

Using Profiles

Once you're comfortable with Simple Mode, you can enable Profiles to unlock the powerful Profiles feature. Profiles allow you to create reusable color palettes with sophisticated mapping options to color many more elements of the VSCode UI beyond the basic elements supported in Simple mode.

Why Use Profiles?

Profiles offer several advantages over simple color rules:

  • Reusability: Define a color palette once and reference it from multiple rules
  • Consistency: Maintain coordinated color schemes across repositories
  • Advanced Mappings: Map individual UI elements to specific palette slots with opacity and adjustments
  • Flexibility: Change an entire theme by updating a single palette definition

The Trade-off: Power vs. Complexity

Profiles are more complex to configure than simple color rules. Instead of specifying colors directly, you create a named palette and then map theme keys to palette slots. This indirection adds flexibility but requires more planning and configuration work.

Start Simple: We strongly recommend becoming proficient with Simple Mode before diving into Profiles. Many users find Simple Mode sufficient for their needs. Only enable Profiles if you need advanced features like palette reuse or per-element color mappings.

Enabling Profiles

To enable Profile mode:

  1. Check the "Enable Profiles" option under Other Settings
  2. The Profiles tab will become active
  3. You can now create profiles and reference them in your rules

For detailed information about creating and using profiles, see the Profiles Guide.

Import & Export

You can export your entire configuration (rules, profiles, and settings) to a JSON file and import it on another machine or share it with teammates. Use the Export Config and Import Config buttons at the bottom of the Rules tab.
