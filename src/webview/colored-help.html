⚡ Colored Keys Guide

The Colored tab is a dynamic overview that automatically shows all VS Code theme keys you've assigned colors to in the current profile. It provides a centralized view of your active customizations, making it easy to review all your color assignments in one place.

What is the Colored Tab?

Unlike the other mapping tabs (Titlebar, Activitybar, Statusbar, etc.) which show all available keys in their respective sections, the Colored tab is a filtered view that displays only the keys you've explicitly customized in the current profile.

⚡ Key Difference: The Colored tab is read-only in the sense that you cannot add new keys to it directly. It automatically populates as you assign colors to keys in other tabs.

How Keys Appear in the Colored Tab

A theme key appears in the Colored tab when you:

  1. Assign a Palette Slot (Primary, Secondary, Accent1-3, etc.) to it
  2. Set a Fixed Color for it
  3. Adjust its Opacity from the default
Note: Keys with no assignment (default state) do NOT appear in the Colored tab. Only keys you've explicitly configured are shown.

Understanding the Tab Badge

The badge on the Colored tab shows the total number of keys you've customized in the current profile:

  • ⚡ 15 means you've assigned colors to 15 different theme keys
  • The badge updates in real-time as you add or remove color assignments
  • Setting a key to "none" still counts as a customization (it's an explicit choice)

Working with Colored Keys

Viewing Your Customizations

Click on the ⚡ Colored tab to see all your active color assignments grouped by section. Keys appear in the same order and organization as the regular tabs, but only showing what you've customized.

Empty Colored Tab

If you haven't assigned any colors yet, the Colored tab displays:

"No color mappings yet. Assign colors in other tabs to see them here."

Navigate to any other tab (Titlebar, Activitybar, etc.) and start assigning colors. Those keys will automatically appear in the Colored tab.

Use Cases and Workflows

Quick Profile Review

When you want to see what you've customized in a profile without browsing through all the sections, the Colored tab gives you an instant overview of all your active changes with hyperlinke to take you directly to each key's source.

Colored vs. Starred Tabs

While both tabs provide filtered views, they serve different purposes:

Colored Tab Starred Tab
Automatic: Shows keys you've assigned colors to Manual: Shows keys you've manually starred
Profile-Specific: Different profiles show different keys Global: Same starred keys across all profiles
Dynamic: Keys appear/disappear based on assignments Persistent: Keys remain until you unstar them
Purpose: See what you've customized Purpose: Quick access to frequently used keys
Example Workflow:
You might have a "Work" profile with 20 customized keys (shown in Colored tab) and 10 starred keys. The starred keys are the ones you customize across all profiles, while the Colored tab shows what you've actually configured in this specific profile.

Technical Details

What Gets Included

The Colored tab includes keys with:

  • Palette slot assignments (Primary, Secondary, etc.)
  • Fixed color values (hex colors like #FF5733)
