Color Report Help

The Color Report shows all VS Code theme keys currently being colored by this extension, along with their applied colors and the source rule or profile applying them.

Understanding the Report

Column Descriptions

Column Description
Theme Key The VS Code theme key being colored (e.g., titleBar.activeBackground). Click on a theme key to jump to its source.
Applied Color The color value currently applied, shown with a color swatch and hex/rgb value.
Applied By The rule or profile that is applying this color. Shows whether it came from a repository rule, branch rule, or profile.

Interactive Hyperlinks

The Color Report includes clickable links that allow you to navigate directly to the source of each color. Click on any Theme Key to jump to its source.

Repository Rule Links

What they look like: Theme keys with colors from repository rules

Where they take you: Clicking on the theme key will:

  • Switch to the Rules tab
  • Scroll to and highlight the specific repository rule that is applying the color
  • If the rule uses a profile, it will take you to the profile instead (see below)

Branch Rule Links

What they look like: Theme keys with colors from branch rules (typically activityBar.* keys)

Where they take you: Clicking on the theme key will:

  • Switch to the Rules tab
  • Scroll to and highlight the specific branch rule that is applying the color
  • If the rule uses a profile, it will take you to the profile instead (see below)

Profile Links

What they look like: Theme keys where the rule references a profile name

Where they take you: Clicking on the theme key will:

  • Switch to the Profiles tab
  • Open the specific profile being used
  • Scroll to and highlight the exact theme key mapping within that profile

This is useful for understanding which palette slot and settings are controlling a particular UI element.

Common Theme Keys

Theme Key UI Element
titleBar.activeBackground Title bar background when window is focused
titleBar.activeForeground Title bar text when window is focused
activityBar.background Activity bar (left sidebar) background
activityBar.foreground Activity bar icon color
statusBar.background Status bar (bottom) background
tab.activeBackground Active editor tab background

Tips

  • Quick Navigation: Use the theme key links to quickly find and edit the source of any color you see in your VS Code window.
  • Debugging: If a color isn't what you expect, check the "Applied By" column to see which rule or profile is responsible.
  • Profile Details: When a profile is used, clicking the theme key will show you the exact mapping, including which palette slot, opacity, and adjustments are being applied.
  • No Colors? If the report is empty, it means no repository or branch rules are currently matching your workspace.
Note: The Color Report only shows colors managed by the Git Repo Window Colors extension. Other color customizations from your VS Code settings or active theme are not included.

Workflow Example

  1. Open the Color Report to see all applied colors
  2. Find a color you want to adjust (e.g., titleBar.activeBackground)
  3. Click on the theme key to jump to its source
  4. Modify the color, palette slot, or mapping as needed
  5. Return to the Color Report to verify your changes
